# Changelog

## [0.1.0-alpha] – 2026-03-21

Initial hackathon prototype of Shroud Protocol.

### Added

- `IdentityRegistry` – on-chain identity commitment store with Merkle root
- `ROSCACircle` – privacy-preserving ROSCA state machine (JOINING → ACTIVE → COMPLETED)
- `MembershipVerifier` – Groth16 proof verifier generated by snarkjs/circom
- `FROSTVerifier` – ECDSA/ecrecover threshold signature verifier (hackathon simplification)
- `membership.circom` – Circom circuit: Merkle inclusion proof with Poseidon hash
- Trusted setup ceremony artifacts (BN254 / pot12)
- Next.js frontend with wallet connection (wagmi + viem)
- In-browser ZK proof generation via snarkjs
- Deployment on Arbitrum Sepolia (chain ID 421614)
- Frontend live at https://shroud-two.vercel.app

### Security

- Low-s enforcement on FROST signatures (EIP-2 malleability prevention)
- `PayoutNotClaimed` guard prevents round progression before recipient claims
- ReentrancyGuard on all ETH-transferring functions
- Nullifier registry prevents double-join across circles

### Known Limitations

- FROST threshold is simulated with a single ECDSA key (not true multi-party)
- Circuit depth fixed at 6 (max 64 members per circle)
- No on-chain randomness for payout order; order follows join sequence
