## PROJECT IDENTITY

- **Name**: Shroud Protocol
- **One-liner**: Privacy-preserving ROSCA (savings circle) using ZK proofs for anonymous membership and FROST threshold signatures for fund custody
- **Hackathon**: Shape Rotator Virtual Hackathon (IC3 / FlashbotsX / Encode Club), March 9–25, 2026
- **Track**: Cryptographic Primitives & Identity
- **Research Papers**: Anonymous Self-Credentials (ePrint 2025/618) + Thetacrypt (arXiv 2502.03247)
- **Chain**: Arbitrum Sepolia (Chain ID 421614)
- **Builder**: Solo (Kunal Drall, 29projects Lab)

## WHAT THIS BUILDS

A working prototype where:
1. Users register identity commitments in an on-chain Merkle tree
2. Users join savings circles by submitting a ZK proof (Groth16) proving they're in the Merkle tree, plus a deterministic nullifier preventing double-joining
3. Members contribute funds each round to a threshold-controlled pool
4. Payouts require a FROST threshold signature (2-of-3) verified on-chain
5. The same user joining different circles produces unlinkable nullifiers (privacy)

**This is NOT**: A full production system. No distributed nodes, no DKG ceremony, no mobile optimization, no mainnet. A hackathon prototype proving the core thesis.

#### Step 1.3: Create the membership circuit

File: `packages/circuits/membership.circom`
```circom
pragma circom 2.1.0;

include "node_modules/circomlib/circuits/poseidon.circom";
include "node_modules/circomlib/circuits/mux1.circom";
include "node_modules/circomlib/circuits/comparators.circom";

// Incremental Merkle tree membership checker using Poseidon
template MerkleTreeChecker(levels) {
    signal input leaf;
    signal input root;
    signal input pathElements[levels];
    signal input pathIndices[levels];

    signal hashes[levels + 1];
    hashes[0] <== leaf;

    component hashers[levels];
    component mux[levels][2];

    for (var i = 0; i < levels; i++) {
        // pathIndices[i] must be 0 or 1
        pathIndices[i] * (1 - pathIndices[i]) === 0;

        // Select left and right inputs based on path
        mux[i][0] = Mux1();
        mux[i][0].c[0] <== hashes[i];
        mux[i][0].c[1] <== pathElements[i];
        mux[i][0].s <== pathIndices[i];

        mux[i][1] = Mux1();
        mux[i][1].c[0] <== pathElements[i];
        mux[i][1].c[1] <== hashes[i];
        mux[i][1].s <== pathIndices[i];

        hashers[i] = Poseidon(2);
        hashers[i].inputs[0] <== mux[i][0].out;
        hashers[i].inputs[1] <== mux[i][1].out;
        hashes[i + 1] <== hashers[i].out;
    }

    root === hashes[levels];
}

template Membership(levels) {
    // Private inputs
    signal input msk;                    // Master secret key
    signal input pathElements[levels];   // Merkle proof siblings
    signal input pathIndices[levels];    // 0=left, 1=right at each level

    // Public inputs
    signal input merkleRoot;             // Current Identity Registry root
    signal input circleId;               // Unique circle identifier

    // Public output
    signal output nullifier;             // Deterministic, circle-specific

    // 1. Compute identity commitment = Poseidon(msk)
    component commitHasher = Poseidon(1);
    commitHasher.inputs[0] <== msk;

    // 2. Verify Merkle tree membership
    component tree = MerkleTreeChecker(levels);
    tree.leaf <== commitHasher.out;
    tree.root <== merkleRoot;
    for (var i = 0; i < levels; i++) {
        tree.pathElements[i] <== pathElements[i];
        tree.pathIndices[i] <== pathIndices[i];
    }

    // 3. Derive deterministic nullifier = Poseidon(msk, circleId)
    component nullHasher = Poseidon(2);
    nullHasher.inputs[0] <== msk;
    nullHasher.inputs[1] <== circleId;
    nullifier <== nullHasher.out;
}

// Main component: depth 6 = 64-member anonymity set
component main {public [merkleRoot, circleId]} = Membership(6);
```

#### Step 2.2: IdentityRegistry.sol

Implement an incremental Merkle tree (Poseidon hash) that:
- Has TREE_DEPTH = 6 (supports 64 leaves)
- Stores filledSubtrees[TREE_DEPTH] for incremental insertion
- Has register(bytes32 commitment) that inserts a leaf and updates the root
- Rejects duplicate commitments
- Emits IdentityRegistered(commitment, leafIndex, newRoot)
- Uses the same Poseidon hash as the Circom circuit (CRITICAL: must match exactly)

**IMPORTANT**: The Poseidon hash implementation in Solidity MUST produce identical outputs to circomlib's Poseidon. Use a well-tested Solidity Poseidon library like:
- https://github.com/iden3/contracts (PoseidonT3 for 2-input hash)
- Or inline the round constants from circomlib

The Merkle tree zeros (empty leaf hashes) must be precomputed using the same Poseidon function. Store them as constants.

#### Step 2.3: ROSCACircle.sol

```solidity
// SPDX-License-Identifier: MIT
pragma solidity ^0.8.24;

import "@openzeppelin/contracts/utils/ReentrancyGuard.sol";
import "./interfaces/IMembershipVerifier.sol";
import "./interfaces/IFROSTVerifier.sol";

contract ROSCACircle is ReentrancyGuard {
    enum State { JOINING, ACTIVE, COMPLETED }

    State public state;
    IMembershipVerifier public immutable verifier;
    IFROSTVerifier public immutable frostVerifier;

    uint256 public immutable memberCount;
    uint256 public immutable contributionAmount;
    uint256 public immutable totalRounds;
    uint256 public immutable roundDuration;
    bytes32 public immutable merkleRoot;
    uint256 public immutable circleId;
    uint256 public immutable thresholdPubKeyX;
    uint256 public immutable thresholdPubKeyY;

    uint256 public currentRound;
    uint256 public joinedCount;
    uint256 public roundDeadline;

    mapping(bytes32 => bool) public usedNullifiers;
    mapping(bytes32 => bool) public isMember;
    bytes32[] public memberList;
    mapping(uint256 => mapping(bytes32 => bool)) public contributed;
    mapping(uint256 => uint256) public roundContribCount;
    mapping(bytes32 => bool) public hasClaimed;

    event MemberJoined(bytes32 indexed pseudonym, uint256 joinedCount);
    event ContributionReceived(uint256 indexed round, bytes32 indexed pseudonym);
    event PayoutDisbursed(uint256 indexed round, address indexed recipient, uint256 amount);
    event RoundAdvanced(uint256 indexed newRound);
    event CircleCompleted();

    constructor(
        address _verifier,
        address _frostVerifier,
        uint256 _memberCount,
        uint256 _contributionAmount,
        uint256 _roundDuration,
        bytes32 _merkleRoot,
        uint256 _circleId,
        uint256 _thresholdPubKeyX,
        uint256 _thresholdPubKeyY
    ) {
        verifier = IMembershipVerifier(_verifier);
        frostVerifier = IFROSTVerifier(_frostVerifier);
        memberCount = _memberCount;
        contributionAmount = _contributionAmount;
        totalRounds = _memberCount; // Each member gets one payout
        roundDuration = _roundDuration;
        merkleRoot = _merkleRoot;
        circleId = _circleId;
        thresholdPubKeyX = _thresholdPubKeyX;
        thresholdPubKeyY = _thresholdPubKeyY;
        state = State.JOINING;
    }

    function joinCircle(
        uint256[2] calldata _pA,
        uint256[2][2] calldata _pB,
        uint256[2] calldata _pC,
        uint256[3] calldata _pubSignals, // [merkleRoot, nullifier, circleId]
        bytes32 _pseudonym
    ) external {
        require(state == State.JOINING, "Not joining");
        require(joinedCount < memberCount, "Full");
        require(!usedNullifiers[bytes32(_pubSignals[1])], "Nullifier used");
        require(_pubSignals[0] == uint256(merkleRoot), "Root mismatch");
        require(_pubSignals[2] == circleId, "Circle mismatch");
        require(!isMember[_pseudonym], "Pseudonym taken");

        // Verify ZK proof
        require(
            verifier.verifyProof(_pA, _pB, _pC, _pubSignals),
            "Invalid proof"
        );

        usedNullifiers[bytes32(_pubSignals[1])] = true;
        isMember[_pseudonym] = true;
        memberList.push(_pseudonym);
        joinedCount++;

        emit MemberJoined(_pseudonym, joinedCount);

        if (joinedCount == memberCount) {
            state = State.ACTIVE;
            currentRound = 1;
            roundDeadline = block.timestamp + roundDuration;
        }
    }

    function contribute(bytes32 _pseudonym) external payable {
        require(state == State.ACTIVE, "Not active");
        require(isMember[_pseudonym], "Not member");
        require(msg.value == contributionAmount, "Wrong amount");
        require(!contributed[currentRound][_pseudonym], "Already contributed");

        contributed[currentRound][_pseudonym] = true;
        roundContribCount[currentRound]++;

        emit ContributionReceived(currentRound, _pseudonym);
    }

    function claimPayout(
        uint256 _sigR,
        uint256 _sigS,
        address payable _recipient
    ) external nonReentrant {
        require(state == State.ACTIVE, "Not active");
        require(currentRound >= 1 && currentRound <= totalRounds, "Invalid round");

        bytes32 pseudonym = memberList[currentRound - 1]; // Simple rotation
        require(!hasClaimed[pseudonym], "Already claimed");

        // Verify FROST threshold signature
        uint256 payoutAmount = contributionAmount * memberCount;
        bytes32 msgHash = keccak256(abi.encodePacked(
            address(this), currentRound, _recipient, payoutAmount
        ));

        require(
            frostVerifier.verify(msgHash, thresholdPubKeyX, thresholdPubKeyY, _sigR, _sigS),
            "Invalid FROST signature"
        );

        hasClaimed[pseudonym] = true;
        (bool sent, ) = _recipient.call{value: payoutAmount}("");
        require(sent, "Transfer failed");

        emit PayoutDisbursed(currentRound, _recipient, payoutAmount);
    }

    function progressRound() external {
        require(state == State.ACTIVE, "Not active");
        require(
            roundContribCount[currentRound] == memberCount ||
            block.timestamp > roundDeadline,
            "Round not complete"
        );

        currentRound++;
        if (currentRound > totalRounds) {
            state = State.COMPLETED;
            emit CircleCompleted();
        } else {
            roundDeadline = block.timestamp + roundDuration;
            emit RoundAdvanced(currentRound);
        }
    }

    function getCircleInfo() external view returns (
        State, uint256, uint256, uint256, uint256, uint256
    ) {
        return (state, joinedCount, memberCount, currentRound, totalRounds, address(this).balance);
    }

    receive() external payable {}
}
```

#### Step 4.4: Proof generation utility
```typescript
// lib/snarkjs-utils.ts
import * as snarkjs from 'snarkjs';

export async function generateMembershipProof(
  msk: bigint,
  pathElements: bigint[],
  pathIndices: number[],
  merkleRoot: bigint,
  circleId: bigint
) {
  const input = {
    msk: msk.toString(),
    pathElements: pathElements.map(e => e.toString()),
    pathIndices: pathIndices,
    merkleRoot: merkleRoot.toString(),
    circleId: circleId.toString(),
  };

  const { proof, publicSignals } = await snarkjs.groth16.fullProve(
    input,
    '/circuits/membership.wasm',    // Serve from public/
    '/circuits/membership.zkey'      // Serve from public/
  );

  // Format for Solidity
  const calldata = await snarkjs.groth16.exportSolidityCallData(proof, publicSignals);
  const [pA, pB, pC, pubSignals] = JSON.parse(`[${calldata}]`);

  return { proof, publicSignals, pA, pB, pC, pubSignals };
}
```

## CRITICAL IMPLEMENTATION NOTES

### Poseidon Hash Compatibility
The #1 integration risk is Poseidon hash mismatch between Circom and Solidity. The on-chain IdentityRegistry MUST use the exact same Poseidon implementation as circomlib. This means:
- Same round constants (from the circomlib `poseidon_constants.json`)
- Same number of full rounds (8) and partial rounds (57 for t=3)
- Same S-box (x^5)
- Same field (BN254 scalar field: 21888242871839275222246405745257275088548364400416034343698204186575808495617)

Test: compute Poseidon([1]) in Circom and in your Solidity contract. They MUST return the same value.

### snarkjs in Browser
- Circuit WASM and zkey files must be served as static assets (put in `public/circuits/`)
- The zkey file for depth-6 Poseidon circuit is ~5-15MB. Consider using streaming if load time is an issue.
- snarkjs.groth16.fullProve() runs entirely client-side — no server needed

### Groth16 Proof Format for Solidity
snarkjs outputs proofs in a format that needs conversion for the auto-generated Solidity verifier. Use `snarkjs.groth16.exportSolidityCallData()` to get the correctly formatted calldata.

### Threshold Signing for Demo
For the hackathon, pre-generate FROST signatures for the demo flow:
1. Use the `frost-secp256k1` Rust crate to run a 2-of-3 key generation
2. Sign a test payout message
3. Hardcode the signature in the frontend for the demo
4. The FROSTVerifier contract verifies it on-chain (proving the architecture works)
5. Document that production would use distributed Thetacrypt nodes

### Gas Optimization Quick Wins
- Use `calldata` not `memory` for proof parameters (saves ~2K gas)
- Use `immutable` for constructor-set values (saves ~2K gas per read)
- Pack struct storage (multiple uint256 values in adjacent slots)
- Use custom errors instead of revert strings (saves ~200 gas per revert)

## IF STUCK: PRIORITIZED FALLBACKS

1. **Circom won't compile** → Check circomlib version compatibility. Use `npm install circomlib@0.5.6` (known stable).
2. **Rust WASM won't build** → Use JavaScript-only approach with `circomlibjs` + `snarkjs`. Still a valid prototype.
3. **FROST verifier too complex** → Use a simple ECDSA signature from a single key. Document that production uses FROST threshold. The architectural design still shows threshold intent.
4. **Poseidon mismatch** → Use keccak256 for Merkle tree in both Circom and Solidity. Less SNARK-friendly (~25K constraints vs ~250) but guaranteed compatibility. Proof gen will be slower (~20s vs ~2s).
5. **Frontend running out of time** → Skip UI polish entirely. A CLI demo (Node.js script) that calls the contracts directly is a valid submission. Record it as the demo video.
6. **Gas too high** → Deploy on Arbitrum Sepolia (10-50x cheaper than L1). If still too expensive for testing, use local Anvil fork for the demo video.
